# 1차 보고서 — 돌핀팟 (DolphinPod)

---

## 1. 과제의 필요성, 배경 및 목표

### 1.1 추진 배경

과학기술정보통신부의 스마트폰 과의존 실태조사(2023)에 따르면, 국내 스마트폰 이용자 10명 중 8명이 자신의 스마트폰 과의존 위험성을 인지하고 있다. 또한 우리 사회의 스마트폰 과의존 문제를 심각하게 인식하는 비율 역시 지속적으로 증가하는 추세이다. 그러나 인식과 실제 행동 변화 사이에는 큰 간극이 존재하며, 그 원인 중 하나는 **사용자가 자신의 구체적인 사용 패턴을 정확히 파악하지 못한다**는 데 있다. "하루에 몇 시간 썼다"는 정보만으로는 언제, 어떤 맥락에서, 어떤 방식으로 스마트폰에 의존하게 되는지를 이해하기 어려우며, 이로 인해 개선 의지가 있어도 구체적인 행동 변화로 이어지지 못하는 경우가 많다.

특히 과의존 사용자군은 일반 사용자군보다 게임, SNS, 영상 등 주요 콘텐츠 이용 비율이 전반적으로 높게 나타나며, 청소년은 게임, 20대는 SNS 이용 비율이 높게 나타나는 경향을 보인다. 이처럼 콘텐츠 중심의 과의존 패턴이 심화됨에 따라, 사용자가 자신의 행동 맥락을 구체적으로 인식하고 자율적으로 개선할 수 있도록 돕는 서비스의 필요성이 커지고 있다.

현재 시장에 출시된 주요 스마트폰 사용 관리 서비스들은 앱별 사용 시간을 단순 집계하거나 일정 시간 초과 시 앱 접근을 강제 차단하는 방식을 채택하고 있다. 이러한 차단 중심의 접근은 즉각적인 사용 억제에는 효과적일 수 있으나, 사용자가 왜 그 앱을 반복적으로 사용하는지, 어떤 상황에서 과사용이 발생하는지에 대한 맥락을 전혀 제공하지 않는다. 결과적으로 차단이 해제된 후 동일한 패턴이 반복되는 한계가 있으며, **사용자의 자율성을 저해하는 문제**가 있다.

행동 변화 연구에 따르면 지속적인 습관 개선을 위해서는 외부 강제가 아닌 **내적 동기와 자기인식**이 핵심이다. 단순한 사용 시간 집계로는 사용자가 스스로 자신의 행동 맥락을 이해하고 개선 방향을 설정하도록 돕기 어렵다. 따라서 사용 데이터를 의미 있는 패턴으로 해석하고, 사용자가 스스로 회고하며 점진적으로 변화할 수 있도록 구조화된 프레임워크와 지속적인 동기 부여 수단을 함께 제공하는 서비스가 필요하다.

### 1.2 타겟 사용자 및 Pain Point

**타겟 사용자**

돌핀팟은 자신의 디지털 행동 패턴을 이해하고, **스스로 더 나은 사용 방식을 만들어가고 싶은 사용자**를 대상으로 한다. 구체적으로는 다음과 같은 특성을 가진 사용자를 주요 타겟으로 설정한다.

- 스마트폰 사용 기록 데이터를 유의미하게 기록하고 싶은 사용자
- 자신의 행동 패턴을 데이터로 이해하고 싶은 사용자
- 강제적인 방식보다는 스스로 개선하고 싶은 사용자
- 소셜 기능을 통한 지속적인 동기 부여를 선호하는 사용자
- 자기관리와 습관 개선에 관심이 있는 사용자

**핵심 Pain Point**

| Pain Point | 설명 |
|-----------|------|
| 사용 패턴 이해 부족 | 자신의 스마트폰 사용 데이터 패턴을 체계적으로 인지하지 못하여, 의미 있는 인사이트나 개선 방향을 얻기 어려움 |
| 습관 지속 동기 부족 | 기록이나 관리를 통해 개선하고 싶지만, 지속적인 동기 및 재미 요소가 부족해 긍정적인 습관 형성에 이르지 못함 |
| 사용자 자율성이 떨어지는 차단 방식 | 기존 서비스들의 강제적인 차단 방식에 대한 거부감으로 습관을 고치려는 시도 자체를 포기하게 됨 |

### 1.3 기존 서비스와의 차별점

**기존 앱 — 단순 사용 시간 관리, 통제 중심**

| 기존 서비스 | 한계 |
|-----------|------|
| Forest | 사용을 강제로 차단 → 사용자 자율성 부족, 거부감 유발 |
| Rize | 단순 사용 시간만 제공, 패턴 분석 없음 → 현재 인식 및 개선 방향성에 대한 정보 없음 |
| Focus Friend | 게임화 요소는 존재하나, 혼자 해야 해서 지속할 동기가 부족함 |

**돌핀팟 — 자기 인식 중심, 소셜 기능 제공**

| 돌핀팟의 차별점 | 내용 |
|--------------|------|
| 높은 사용자 자율성 제공 | 사용 패턴 분석 제공 후 사용자 스스로 KPT 회고·개선 → 자율적인 패턴 선택 유도 |
| AI 기반 사용 패턴 세션화 | 단순 사용 시간 집계뿐만 아니라, AI 기반 사용 패턴 분석 및 특이 행동 탐지를 제공 |
| 지속적인 동기 부여 | 게이미피케이션으로 동기를 유지하고, 그룹 챌린지로 장기적인 참여를 강화 |

### 1.4 과제 목표

돌핀팟(DolphinPod)은 Android UsageStatsManager API를 통해 수집한 스마트폰 사용 데이터를 세션 단위로 구조화하고, AI가 해당 데이터를 분석하여 특이 사용 패턴 5가지 후보를 자연어 문장으로 생성해 사용자에게 제시하는 Android 어플리케이션이다. 단순히 사용 시간을 수치로 보여주는 데 그치지 않고, "늦은 밤 SNS를 반복적으로 확인했어요"와 같이 사용자가 자신의 행동을 직관적으로 인식할 수 있는 형태로 데이터를 전달하는 것을 목표로 한다.

사용자는 AI가 제시한 5가지 패턴 후보 중 오늘 기록으로 남기고 싶은 항목을 직접 선택하고, KPT(Keep/Problem/Try) 프레임워크에 따라 이어가고 싶은 패턴(K), 고치고 싶은 패턴(P), 개선하고 싶은 패턴(T)으로 분류하여 회고를 작성한다. 이렇게 축적된 KPT 회고 데이터와 온보딩 시 수집한 사용자 초기 정보를 결합하여 AI가 데일리·주간 개인화 리포트를 생성하며, 사용자는 단순 데이터 열람을 넘어 자신의 행동 패턴에 대한 구체적인 인사이트를 얻을 수 있다.

자기인식만으로는 습관 변화가 오래 지속되기 어렵다는 점을 고려하여, 돌핀팟은 게이미피케이션과 소셜 기능을 통해 지속적인 동기 부여 구조를 마련한다. 체크인 및 그룹 챌린지 달성 시 코인을 획득하고 이를 통해 개인 캐릭터를 커스텀할 수 있으며, 일정 기준을 달성하면 업적 뱃지가 부여된다. 친구와 함께하는 그룹 챌린지와 응원 주고받기 기능 등의 소셜 기능은 사용자가 혼자가 아닌 커뮤니티 안에서 변화를 만들어가는 경험을 제공하며, 스마트폰이 주는 자극을 앱 내 긍정적 자극으로 대체하는 구조를 지향한다.

## 2. 과제의 산출물

### 2.1 Android 어플리케이션 (DolphinPod)

데일리 체크인, AI 리포트, 소셜 기능, 게이미피케이션을 포함한 실사용 가능한 Android 앱을 개발하여 Google Play Store에 등록한다.

---

#### 2.1.1 데일리 체크인

매일 지정 시간에 푸시 알림을 발송하고, 당일 사용 데이터를 세션 단위로 전처리한 후 AI가 특이 사용 패턴 5가지 후보를 자연어 문장으로 제시한다. 사용자는 KPT(Keep/Problem/Try) 형식으로 패턴을 선택하여 회고를 기록한다.

**주요 기능**

- 푸시 알림: 매일 사용자가 지정한 시간에 체크인 알림 발송
- 당일 사용 데이터를 세션 단위로 전처리 후 AI 호출
- AI 자연어 기반 문장 생성 후 5개의 특이 사용 패턴 후보 제시
- KPT 패턴 선택 및 기록
  - K(Keep): 이어가고 싶은 패턴
  - P(Problem): 고치고 싶은 패턴
  - T(Try): 개선하고 싶은 패턴

> **[이미지: 데일리 체크인 UI 화면]**
>
> *(체크인 패턴 목록 화면 / KPT 선택 화면)*

---

#### 2.1.2 AI 개인화 리포트

데일리 리포트와 주간 리포트로 구성되며, 두 리포트 모두 캘린더에서 조회 가능하다.

**데일리 리포트**

- 사용자가 선택한 KPT 패턴 / 사용 흐름 / 주 사용 카테고리 정보 제공
- 당일 KPT 패턴과 온보딩 정보를 기반으로 AI 코멘트 생성

**주간 리포트**

- 데일리 리포트와 온보딩 정보를 기반으로 생성
- 점수 / 추세분석 / 체크인 참여 횟수 정보 제공
- AI 기반 주간 코멘트 생성

> **[이미지: 데일리 리포트 UI 화면]**
>
> *(사용 흐름 그래프, 주 사용 카테고리, AI 코멘트 포함)*

> **[이미지: 주간 리포트 UI 화면]**
>
> *(점수 추세 분석, 체크인 참여 현황, AI 주간 코멘트 포함)*

> **[이미지: 캘린더 조회 화면]**
>
> *(일별 체크인 기록 및 리포트 조회 캘린더)*

---

#### 2.1.3 소셜 기능

Google OAuth 기반 계정 연동을 바탕으로 소셜 기능을 제공한다.

- 친구 추가, 삭제, 신청 등 기본 친구 커넥션 기능
- 서로 응원 알림 주고받기 (응원 메시지는 고정 문구에서 랜덤 발송)
- 친구 사용자들과 일주일 단위 그룹 챌린지 기능 (체크인 기록, 리포트 조회, 주간 리포트 생성 성공 등)
- 그룹 챌린지 달성 보상으로 골드/경험치를 난이도 및 지속 일수에 따라 차등 지급
- 그룹 상태 화면에서 다른 사용자들의 메인 캐릭터 확인 가능 (같은 바다에서 유영하는 UI로 연대감 제공)

> **[이미지: 소셜 기능 UI 화면]**
>
> *(친구 목록 화면 / 그룹 챌린지 바다 화면)*

#### 2.1.4 게이미피케이션

- DolphinPod만의 스토리와 세계관이 담긴 게임 엔진 탑재

- 체크인 및 그룹 챌린지 보상으로 코인 획득 → 개인 캐릭터(캐릭터·액세서리·배경·이펙트) 커스텀 가능

> **[이미지: 게이미피케이션 UI 화면]**
>
> *(캐릭터 커스텀 화면 / 업적 달성 뱃지 화면)*

### 2.2 AI 파이프라인 — 프롬프트 출력 결과 검증

GPT-4o-mini 기반 체크인 파이프라인 및 Claude Sonnet 기반 리포트 파이프라인, 2단계 QA 시스템 소스코드를 GitHub에 공개한다.

**체크인 파이프라인 출력 결과**

- 사용자의 기기 사용 데이터(앱별 사용 시간, 세션 패턴)를 입력으로 받아 5개의 특이 패턴 후보를 자연어로 생성
- Deterministic QA 검증(스키마·수치·금지어)과 LLM Judge를 통한 2차 품질 검증 수행

> **[이미지: AI 체크인 파이프라인 프롬프트 출력 결과]**
>
> *(Deterministic QA 검증 화면 / 패턴 후보 JSON 출력 결과)*

**리포트 파이프라인 출력 결과**

- KPT 기록 + 사용 데이터 + RAG 기반 연구 지식베이스를 결합하여 개인화 리포트 생성
- Claude Sonnet 기반 리포트 작성자 → 리포트 검수자 → 최종 맞춤형 리포트 발행

> **[이미지: AI 리포트 파이프라인 프롬프트 출력 결과]**
>
> *(daily-report API 응답 JSON / AI 코멘트 생성 결과)*

**Android 기기 사용 데이터 수집 구현 결과**

수집되는 데이터 항목은 다음과 같다.

- 기기 잠금 해제 횟수
- 사용 권한 허용 상태
- 특정 앱 방문 횟수
- 앱별 총 사용 시간
- 특정 앱 최장 연속 사용 시간

> **[이미지: Android 기기 사용 데이터 수집 화면]**
>
> *(실기기 테스트 화면 — 앱별 사용 시간 및 방문 횟수 수집 결과)*

**FE-BE-DB 파이프라인 연동 결과**

- 사용자의 기기 사용 데이터(앱별) 수집 및 Supabase DB 저장
- FE-BE-DB 파이프라인 연동 완료 (실시간 조회 가능)
- PostgreSQL 기반 Supabase DB 구축 완료

> **[이미지: FE-BE-DB 연동 테스트 화면]**
>
> *(API 요청/응답 로그 또는 Supabase DB 데이터 조회 화면)*

## 3. 과제의 SW 구조

*(SW 구조도)*

돌핀팟의 SW 구조는 크게 **클라이언트(프론트엔드)**, **서버(백엔드 + AI 파이프라인)**, **데이터베이스**, **AI API**, **CI/CD 자동화**로 구성된다.

### 3.1 클라이언트 (React Native)

사용자가 직접 상호작용하는 Android 앱 레이어로, 다음 기능을 수행한다.

- 구글 로그인 / 회원 가입
- 온보딩 초기 설정 데이터 입력
- 체크인 패턴 KPT 선택
- 리포트 조회 (데일리 / 주간 / 캘린더)
- 그룹 생성 & 챌린지 참여
- 소셜 기능 (친구 추가 등)

주요 기술 요소:

- 프레임워크: React Native (기능별 화면 전체 렌더링)
- Google OAuth (로그인 / 인증)
- Zustand (상태 관리 라이브러리 → 사용자 세션 관리)

클라이언트는 REST API를 통해 서버와 통신하며, 초기 설정 데이터 / KPT 선택 데이터 / 기기 사용 데이터(UsageStats)를 전달한다.

### 4.1 Pain Point — Solution 매핑

돌핀팟이 제안하는 Solution은 3가지 Pain Point에 대응하는 4가지 핵심 Solution으로 구성된다.

| Pain Point | Solution |
|-----------|----------|

| 사용 패턴 이해 부족 | ② 사용 데이터 + 개인화 정보 기반 AI 리포트 제공 |

### 4.2 서비스 핵심 목표 및 워크플로우

> 스마트폰 사용 데이터를 수집하고 AI로 분석하여 포착된 패턴을 제공함으로써,
> **사용자가 자신의 패턴을 인식하고 습관을 개선할 수 있게 돕는다.**

서비스 전체는 다음 4단계 워크플로우로 작동한다.

---

**1단계 — 사용 데이터 수집 및 로컬 전처리**

- Android UsageStats API를 사용하여 앱별 사용 시간, 실행 횟수 등 수집
- 사용 데이터를 snapshot으로 가공하여 시간대별 사용 패턴 등의 분석 가능한 데이터셋으로 재구성

**2단계 — AI 사용 패턴 분석**

- LLM 기반 특이 패턴 감지
- KPT를 사용자가 스스로 평가하여 매일의 기록 완성
- 체크인 응답(KPT)과 결합하여 맥락 분석

**3단계 — AI 리포트 생성**

- 분석된 리포트 생성 결과 변환
- 일일/주간 개인화 피드백 제공
- 사용자 초기 설정 정보 반영
- 코드 1차 검수, AI 검수자를 통한 2차 검수로 안정된 응답 제공

**4단계 — 게임화 요소**

- 프로필/배경 커스텀
- 소셜 기능
- 업적 달성 시 보상 시스템 → 체크인 동기 부여

---

### 4.3 AI 멀티 에이전트 파이프라인 상세 구조

돌핀팟의 핵심 기술 구조는 3단계 워크플로우(WF1~WF3)로 구성된 AI 멀티 에이전트 파이프라인이다. AI 품질 검증 및 신뢰도를 위한 멀티 에이전트 설계를 통해 **사용자별 개인화된 리포트를 생성**한다.

---

**WF1 — Raw data → Snapshot**

원시 로그 데이터(세션 / 언락 / 사용량)를 수집하고 사용량을 계산하여 Daily Snapshot을 생성한다.

- 입력: 원시 로그 데이터 (세션 / 언락 / 사용량)
- 처리: 사용량 계산
- 출력: Daily Snapshot 생성 → WF2로 전달

---

**WF2 — 체크인 패턴 후보 생성**

Daily Snapshot을 기반으로 AI가 특이 패턴을 탐지하고, 품질 검증을 거쳐 사용자에게 패턴 후보를 제시한다.

- **체크인 생성자 (GPT-4o-mini)**: Snapshot 데이터 기반 패턴 후보 5개 생성
- **체크인 검수자**: 결정론적 검증(스키마·수치·금지어) + LLM Judge 정성 판단으로 품질 검증
- **유저 선택지 제공**: 검증된 패턴 후보를 사용자에게 제시 → 사용자가 패턴 선택 + KPT 기록 완성

---

**WF3 — RAG 기반 리포트 생성**

사용자의 KPT 기록과 RAG 지식베이스를 결합하여 근거 있는 개인화 리포트를 생성하고, 검수 후 최종 발행한다.

- **RAG Engine — 근거 검색 쿼리 생성자**: 사용자 데이터 기반 검색 쿼리 생성
- **Supabase Vector Store**: pgvector 기반 유사 문서 검색 및 Context 전달
- **리포트 작성자 (Claude Sonnet)**: RAG Context + KPT 기록 + 사용자 설정 정보를 결합하여 개인화 리포트 작성
- **리포트 검수자**: 생성된 리포트 품질 검증
- **최종 맞춤형 리포트 발행**: 검수 완료된 리포트를 사용자에게 제공

> 사용자 환경설정(User Configs)은 **WF2 체크인 생성자**와 **WF3 쿼리 생성자**에 공통 적용된다.
