# 1차 보고서 — 돌핀팟 (DolphinPod)

---

## 1. 과제의 필요성, 배경 및 목표

### 1.1 추진 배경

과학기술정보통신부의 스마트폰 과의존 실태조사에 따르면, 국내 스마트폰 이용자 10명 중 8명이 자신의 스마트폰 과의존 위험성을 인지하고 있다. 그러나 인식과 실제 행동 변화 사이에는 큰 간극이 존재하며, 그 원인 중 하나는 **사용자가 자신의 구체적인 사용 패턴을 정확히 파악하지 못한다**는 데 있다. "하루에 몇 시간 썼다"는 정보만으로는 언제, 어떤 맥락에서, 어떤 방식으로 스마트폰에 의존하게 되는지를 이해하기 어려우며, 이로 인해 개선 의지가 있어도 구체적인 행동 변화로 이어지지 못하는 경우가 많다.

현재 시장에 출시된 주요 스마트폰 사용 관리 서비스들은 앱별 사용 시간을 단순 집계하거나 일정 시간 초과 시 앱 접근을 강제 차단하는 방식을 채택하고 있다. 이러한 차단 중심의 접근은 즉각적인 사용 억제에는 효과적일 수 있으나, 사용자가 왜 그 앱을 반복적으로 사용하는지, 어떤 상황에서 과사용이 발생하는지에 대한 맥락을 전혀 제공하지 않는다. 결과적으로 차단이 해제된 후 동일한 패턴이 반복되는 한계가 있으며, **사용자의 자율성을 저해하는 문제**가 있다.

행동 변화 연구에 따르면 지속적인 습관 개선을 위해서는 외부 강제가 아닌 **내적 동기와 자기인식**이 핵심이다. 단순한 사용 시간 집계로는 사용자가 스스로 자신의 행동 맥락을 이해하고 개선 방향을 설정하도록 돕기 어렵다. 따라서 사용 데이터를 의미 있는 패턴으로 해석하고, 사용자가 스스로 회고하며 점진적으로 변화할 수 있도록 구조화된 프레임워크와 지속적인 동기 부여 수단을 함께 제공하는 서비스가 필요하다.

### 1.2 필요성

기존 서비스와 돌핀팟의 차별점을 분석하면 다음과 같다.

| 구분 | 기존 서비스 | 돌핀팟 |
|------|------------|--------|
| 인지 행동 치료 개선 앱 (MindDoc, Wysa, Woebot) | 대화형 상담 / 감정 추적 / 리포트 제공, 시각적 몰입 요소 부재 | - |
| 정신 건강 관리 앱 (Quabble) | 멘탈 관리 / 감정 기록, 사용자가 데이터를 수동 입력 | - |
| 집중 유도 앱 (Forest, Flipd, Opal) | 몰입 요소 제공 / 타이머 / 앱 차단, 행동 개선·지속적 동기 유도 부족 | - |
| **돌핀팟** | - | 개인화 리포트 + 행동 데이터 자동 수집·분석 + 행동 개선에 따른 시각적 보상 → 건강한 피드백으로 전환 |

위 표와 같이 기존 서비스들은 개인화 리포트, 자동 행동 데이터 수집·분석, 지속적 동기 부여 요소 중 일부만을 제공하는 반면, 돌핀팟은 이를 통합적으로 제공한다.

### 1.3 목표

돌핀팟(DolphinPod)은 Android UsageStatsManager API를 통해 수집한 스마트폰 사용 데이터를 세션 단위로 구조화하고, AI가 해당 데이터를 분석하여 특이 사용 패턴 5가지 후보를 자연어 문장으로 생성해 사용자에게 제시하는 Android 어플리케이션이다. 단순히 사용 시간을 수치로 보여주는 데 그치지 않고, "늦은 밤 SNS를 반복적으로 확인했어요"와 같이 사용자가 자신의 행동을 직관적으로 인식할 수 있는 형태로 데이터를 전달하는 것을 목표로 한다.

사용자는 AI가 제시한 5가지 패턴 후보 중 오늘 기록으로 남기고 싶은 항목을 직접 선택하고, KPT(Keep/Problem/Try) 프레임워크에 따라 이어가고 싶은 패턴(K), 고치고 싶은 패턴(P), 개선하고 싶은 패턴(T)으로 분류하여 회고를 작성한다. 이렇게 축적된 KPT 회고 데이터와 온보딩 시 수집한 사용자 초기 정보를 결합하여 AI가 데일리·주간 개인화 리포트를 생성하며, 사용자는 단순 데이터 열람을 넘어 자신의 행동 패턴에 대한 구체적인 인사이트를 얻을 수 있다.

자기인식만으로는 습관 변화가 오래 지속되기 어렵다는 점을 고려하여, 돌핀팟은 게이미피케이션과 소셜 기능을 통해 지속적인 동기 부여 구조를 마련한다. 체크인 및 그룹 챌린지 달성 시 코인을 획득하고 이를 통해 개인 캐릭터를 커스텀할 수 있으며, 일정 기준을 달성하면 업적 뱃지가 부여된다. 친구와 함께하는 그룹 챌린지와 응원 주고받기 기능 등의 소셜 기능은 사용자가 혼자가 아닌 커뮤니티 안에서 변화를 만들어가는 경험을 제공하며, 스마트폰이 주는 자극을 앱 내 긍정적 자극으로 대체하는 구조를 지향한다.

## 2. 과제의 산출물

### 2.1 Android 어플리케이션 (DolphinPod)

데일리 체크인, AI 리포트, 소셜 기능, 게이미피케이션을 포함한 실사용 가능한 Android 앱을 개발하여 Google Play Store에 등록한다.

#### 주요 기능별 산출물

**데일리 체크인**

매일 지정 시간에 푸시 알림을 발송하고, 당일 사용 데이터를 세션 단위로 전처리한 후 AI가 특이 사용 패턴 5가지 후보를 자연어 문장으로 제시한다. 사용자는 KPT(Keep/Problem/Try) 형식으로 패턴을 선택하여 회고를 기록한다.

- 푸시 알림 기능: 매일 사용자가 지정한 시간에 체크인 알림
- 당일 사용 데이터를 세션 단위로 전처리 후 AI 호출
- AI는 자연어 기반 문장 생성 후 5개의 특이 사용 패턴 후보 제시
- KPT 패턴 후보에 대해 선택으로 기록으로 남기고 싶은 패턴 저장
  - K(Keep): 이어가고 싶은 패턴
  - P(Problem): 고치고 싶은 패턴
  - T(Try): 개선하고 싶은 패턴

**AI 개인화 리포트**

리포트는 데일리 / 주간 리포트로 나뉘며, 두 리포트 모두 캘린더에서 조회 가능하다.

- 데일리 리포트: 사용자가 선택한 KPT 패턴 / 사용 흐름 / 주 사용 카테고리 정보 제공, 당일 KPT 패턴과 온보딩 정보를 기반으로 AI 기반 코멘트 생성
- 주간 리포트: 데일리 리포트와 온보딩 정보를 기반으로 생성, 점수 / 추세분석 / 체크인 참여 횟수 정보 제공 및 AI 기반 코멘트 생성

**소셜 기능**

- Google OAuth 기반 계정 연동을 바탕으로 소셜 기능 제공
- 친구 추가, 삭제, 신청 등의 기본적인 친구 커넥션 기능 제공 (서로 응원 알림 주고받기 가능, 응원 메시지는 고정되어 있고 랜덤으로 발송)
- 친구 사용자들과 일주일 단위로 그룹 챌린지 기능 제공 (체크인 기록, 리포트 조회, 주간 리포트 생성 성공 등)
- 그룹 챌린지 달성에 대한 보상으로 골드나 경험치를 난이도나 지속 일수에 따라 차등 지급함으로써 사용자의 습관 개선 동기 유도
- 그룹 상태 화면에서 다른 사용자들의 메인 캐릭터를 확인할 수 있으며, 같은 바다에서 유영하는 느낌의 UI로 습관 개선을 함께하는 연대감 제공

**게이미피케이션**

- DolphinPod만의 스토리와 세계관이 담긴 게임 엔진
- 꾸준한 사용 습관 개선을 위한 일정 기준치 달성 시 재밌는 타이틀이 담긴 업적 달성 뱃지 부여
- 체크인 및 그룹 챌린지 보상으로 코인을 획득하면 개인 캐릭터 커스텀 가능

### 2.2 핵심 소스코드

GPT-4o-mini 기반 체크인 파이프라인 및 Claude 기반 리포트 파이프라인, 2단계 QA 시스템 소스코드를 GitHub에 공개한다.

### 2.3 프로그램 저작권 등록

개발 완료된 소프트웨어에 대해 한국저작권위원회에 프로그램 저작권을 등록하여, 본 프로젝트의 기술적 독창성을 공식적으로 보호하고 향후 서비스 확장의 법적 기반을 마련한다.

## 3. 과제의 SW 구조

*(SW 구조도)*

돌핀팟의 SW 구조는 크게 클라이언트(프론트엔드), 서버(백엔드 + AI 파이프라인), 데이터베이스, AI API, CI/CD 자동화로 구성된다.

### 3.1 클라이언트 (React Native)

사용자가 직접 상호작용하는 Android 앱 레이어로, 다음 기능을 수행한다.

- 구글 로그인 / 회원 가입
- 온보딩 초기 설정 데이터 입력
- 체크인 패턴 KPT 선택
- 리포트 조회
- 그룹 생성 & 챌린지 참여
- 소셜 기능 (친구 추가 등)

주요 기술 요소:

- 프레임워크: React Native (기능별 화면 전체 렌더링)
- Google OAuth (로그인 / 인증)
- Zustand (상태 관리 라이브러리 → 사용자 세션 관리)
- UsageStatsManager (앱 사용 데이터 수집)

클라이언트는 REST API를 통해 서버와 통신하며, 초기 설정 데이터 / KPT 선택 데이터 / 기기 사용 데이터(UsageStats)를 전달한다.

### 3.2 서버 (AWS EC2 — FastAPI)

서버는 AWS EC2 위에서 FastAPI로 구동되며, 기능 서버(BE)와 AI Agent 파이프라인으로 구분된다.

**기능 서버 (BE)**

| 모듈 | 주요 역할 |
|------|----------|
| Onboarding | 사용자 프로필 생성, 사용 목표 설정, 구글 로그인 처리, 알림/데이터 접근 관리 |
| DashBoard | 사용 데이터 집계 및 반환, 사용 패턴 시각화, 주요 사용 앱 순위 제공, 체크인/리포트 이력 데이터 |
| Social | 그룹 챌린지 생성·참여·조회, 그룹 내 공유 데이터 제공, 챌린지 완료 조건 검증, 보상 지급 트리거 발생 |
| Gamification | 활동 기반 보상 시스템 로직 (업적 및 뱃지 컬렉션), 캐릭터 및 바다 커스텀 관리 |

**AI Agent 파이프라인**

| 파이프라인 | 역할 |
|-----------|------|
| Checkin | 사용자 정보 + 당일 사용 데이터 분석, 유의미한 패턴 리스트 추출 |
| Report | 데일리 리포트: 사용자 정보 + KPT 기록, 주간 리포트: 데일리 리포트 경향 분석 |
| 품질 검증 AI | 생성된 체크인, 리포트 검수 |

### 3.3 데이터베이스 (Supabase / PostgreSQL)

| 구성 요소 | 역할 |
|----------|------|
| Supabase (PostgreSQL 호스팅 인프라) | PostgreSQL 호스팅, 실시간 로컬 세션 저장 |
| PostgreSQL | 메인 DB — 데이터 관계 및 구조 관리 (사용자 정보, 체크인/리포트 기록, 챌린지 데이터 등) |
| pgvector (PostgreSQL의 확장 플러그인) | RAG 지식 베이스 데이터 저장, 리포트 작성 시 검색 후 유사도 저장 |

### 3.4 AI API

| API | 역할 |
|-----|------|
| GPT-4o-mini (OpenAI) | 체크인 질문 생성 |
| Claude Sonnet (Anthropic) | 리포트 생성 |

### 3.5 CI/CD 자동화

GitHub Actions + PromptFoo CI를 활용하여 AI 프롬프트 검증 에이전트(프롬프트 회귀 테스트)를 자동화하고, AWS EC2 서버에 배포한다.

## 4. 과제에서 제안하는 Solution의 구조

돌핀팟이 제안하는 Solution은 크게 4가지 Pain Point에 대응하는 4가지 핵심 Solution으로 구성된다.

| Pain Point | Solution |
|-----------|----------|
| 사용 패턴 이해 부족 | ① 데일리 체크인 기반 사용 패턴 인식 |
| 습관 지속 동기 부족 | ② 사용 데이터 + 개인화 정보 기반 AI 리포트 제공 |
| 습관 지속 동기 부족 | ③ 소셜 기능을 통한 지속 동기 강화 |
| 사용자 자율성이 떨어지는 차단 방식 | ④ 게이미피케이션 기반 자극 대체, 자기 인식 |

### 4.1 Solution 작동 흐름

돌핀팟의 전체 서비스는 다음 4단계로 작동한다.

**1단계 — 사용 데이터 수집 및 로컬 전처리**

- Android UsageStats API를 사용하여 앱별 사용 시간, 실행 횟수 등 수집
- 사용 데이터를 snapshot으로 가공 (시간대별 사용 패턴 등의 분석 가능한 데이터셋으로 재구성)

**2단계 — AI 사용 패턴 분석**

- LLM 기반 특이 패턴 감지
- KPT를 사용자가 스스로 평가하여 매일의 기록 완성
- 체크인 응답(KPT)과 결합하여 맥락 분석

**3단계 — AI 리포트 생성**

- 분석된 리포트 생성 결과 변환
- 일일/주간 개인화 피드백 제공
- 사용자 초기 설정 정보 반영
- 코드 1차 검수, AI 검수자를 통한 2차 검수로 안정된 응답 제공

**4단계 — 게임화 요소**

- 프로필/배경 커스텀
- 소셜 기능
- 업적 달성 시 보상 시스템 → 체크인 동기 부여

### 4.2 AI 파이프라인 상세 구조 (기술 작동 흐름)

**WF1 — Raw data → Snapshot**

- 원시 로그 데이터 (세션 / 언락 / 사용량) 수집
- 사용량 계산 후 Daily Snapshot 생성

**WF2 — 체크인 패턴 후보 생성**
