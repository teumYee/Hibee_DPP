## 2. Project-Summary (과제 요약)

### 2.1 문제 정의

과학기술정보통신부의 스마트폰 과의존 실태조사(2023)에 따르면, 국내 스마트폰 이용자 10명 중 8명이 자신의 스마트폰 과의존 위험성을 인지하고 있다. 그러나 인식과 실제 행동 변화 사이에는 큰 간극이 존재한다.

그 핵심 원인은 세 가지로 정리된다.

**① 사용 패턴 이해 부족**
자신의 스마트폰 사용 데이터 패턴을 체계적으로 인지하지 못하여, "하루에 몇 시간 썼다"는 수치 외에 언제·어떤 맥락에서·어떤 방식으로 의존하게 되는지에 대한 의미 있는 인사이트나 개선 방향을 얻기 어렵다.

**② 습관 지속 동기 부족**
기록이나 관리를 통해 개선하고 싶지만, 지속적인 동기 및 재미 요소가 부족해 긍정적인 습관 형성에 이르지 못한다.

**③ 사용자 자율성이 떨어지는 차단 방식**
기존 서비스들의 강제적인 차단 방식에 대한 거부감으로 습관을 고치려는 시도 자체를 포기하게 된다. 차단이 해제된 후에도 동일한 패턴이 반복되는 근본적인 한계가 있다.

### 2.2 기존연구와의 비교

현재 시장에 출시된 주요 스마트폰 사용 관리 서비스들은 각각 일부 기능만을 제공하며 다음과 같은 한계를 가진다.

| 서비스 | 제공 기능 | 한계 |
|--------|-----------|------|
| Forest | 앱 강제 차단 / 집중 타이머 | 사용자 자율성 부족, 차단 해제 후 동일 패턴 반복 |
| Rize | 사용 시간 집계 | 단순 수치만 제공, 패턴 분석 및 개선 방향성 없음 |
| Focus Friend | 게임화 요소 (혼자) | 혼자 해야 하므로 지속할 동기 부족 |
| MindDoc / Wysa / Woebot | 대화형 상담 / 감정 추적 / 리포트 | 시각적 몰입 요소 부재, 사용 데이터 자동 수집 없음 |
| Quabble | 멘탈 관리 / 감정 기록 | 사용자가 데이터를 수동으로 입력해야 함 |

**돌핀팟의 차별점:**

| 차별점 | 내용 |
|--------|------|
| 높은 사용자 자율성 | 사용 패턴 분석 후 사용자 스스로 KPT 회고·개선 → 자율적인 패턴 선택 유도 |
| AI 기반 사용 패턴 세션화 | 단순 사용 시간 집계를 넘어, AI 기반 패턴 분석 및 특이 행동 탐지 제공 |
| 지속적인 동기 부여 | 게이미피케이션으로 동기를 유지하고, 그룹 챌린지로 장기적인 참여를 강화 |

### 2.3 제안 내용

돌핀팟(DolphinPod)은 Android UsageStatsManager API로 수집한 스마트폰 사용 데이터를 세션 단위로 구조화하고, AI가 특이 사용 패턴 5가지 후보를 자연어 문장으로 생성해 사용자에게 제시하는 AI 기반 Android 어플리케이션이다.

사용자는 AI가 제시한 패턴 후보 중 기록으로 남기고 싶은 항목을 선택하고, KPT(Keep/Problem/Try) 프레임워크에 따라 패턴을 분류하여 회고를 작성한다. 축적된 KPT 데이터와 온보딩 초기 정보를 결합하여 AI가 데일리·주간 개인화 리포트를 생성하며, 게이미피케이션과 소셜 기능으로 지속적인 동기 부여 구조를 마련한다.

핵심 철학은 **"차단이 아닌 자기인식"**이다.

### 2.4 기대 효과 및 의의

- **사용자 측면:** 스마트폰 사용 데이터 분석을 통해 강제적인 차단이 아닌 자율적 사용 습관 개선을 지원함. KPT 회고 프레임워크와 AI 개인화 리포트를 결합하여 단발성 인식에 그치지 않고 지속적인 자기 모니터링 습관을 형성할 수 있음.
- **사회적 측면:** 디지털 웰빙 앱 시장에서 차별화된 자기인식 기반 서비스로 출시하여, 스마트폰 과의존 문제를 스스로 해결하고자 하는 사용자를 지원함. 학교의 미디어 리터러시 교육 도구나 청소년 스마트폰 과의존 예방 사업과 연계하여 사회적 가치를 창출할 수 있음.
- **확장 가능성:** 수면·운동·식습관 등 타 생활습관 영역으로 AI 개인화 행동 변화 플랫폼을 확장할 수 있음.
- **교육적 측면:** 실서비스 수준의 AI 파이프라인(LLM + RAG + QA 시스템) 설계 및 구현 경험을 통해, 이론으로만 접하던 LLM 응용 개발과 프롬프트 엔지니어링, 품질 검증 체계를 실무적으로 습득할 수 있음.

### 3.2 전체 시스템 구성

*(SW 구조도)*

돌핀팟의 전체 시스템은 **클라이언트**, **서버(BE + AI 파이프라인)**, **데이터베이스**, **AI API**, **CI/CD** 5개 레이어로 구성된다.

#### 클라이언트 (React Native)

| 구성 요소 | 역할 |
|-----------|------|
| React Native | 기능별 화면 전체 렌더링 |
| Google OAuth | 로그인 / 인증 |
| Zustand | 상태 관리 → 사용자 세션 관리 |
| UsageStatsManager | 앱 사용 데이터 수집 |

클라이언트는 REST API를 통해 서버와 통신하며 초기 설정 데이터, KPT 선택 데이터, 기기 사용 데이터(UsageStats)를 전달한다.

#### 서버 (AWS EC2 — FastAPI)

**기능 서버 (BE)**

| 모듈 | 주요 역할 |
|------|-----------|
| Onboarding | 사용자 프로필 생성, 사용 목표 설정, 구글 로그인 처리, 알림/데이터 접근 관리 |
| DashBoard | 사용 데이터 집계 및 반환, 사용 패턴 시각화, 주요 사용 앱 순위 제공, 체크인/리포트 이력 |
| Social | 그룹 챌린지 생성·참여·조회, 그룹 내 공유 데이터 제공, 챌린지 완료 조건 검증, 보상 지급 트리거 |
| Gamification | 활동 기반 보상 시스템, 업적·뱃지 컬렉션, 캐릭터·바다 커스텀 관리 |

**AI Agent 파이프라인**

| 파이프라인 | 역할 |
|-----------|------|
| Checkin | 사용자 정보 + 당일 사용 데이터 분석 → 유의미한 패턴 리스트 추출 |
| Report | 데일리: KPT 기록 기반 생성 / 주간: 데일리 리포트 경향 분석 기반 생성 |
| 품질 검증 AI | 생성된 체크인·리포트 검수 |

| 구성 요소 | 역할 |

### 3.3 주요 엔진 및 기능 설계

#### AI 멀티 에이전트 파이프라인 (핵심 Solution 구조)

돌핀팟의 핵심 기술은 3단계 워크플로우(WF1~WF3)로 구성된 AI 멀티 에이전트 파이프라인이다. AI 품질 검증 및 신뢰도를 위한 멀티 에이전트 설계를 통해 사용자별 개인화된 리포트를 생성한다.

> ※ 사용자 환경설정(User Configs)은 WF2 체크인 생성자와 WF3 쿼리 생성자에 공통 적용된다.

---

**WF1 — Raw data → Snapshot**

- 입력: 원시 로그 데이터 (세션 / 언락 / 사용량)
- 처리: 사용량 계산
- 출력: Daily Snapshot 생성 → WF2로 전달

---

**WF2 — 체크인 패턴 후보 생성**

- **체크인 생성자 (GPT-4o-mini):** Snapshot 데이터 기반 패턴 후보 5개 생성
- **체크인 검수자:** 결정론적 검증(스키마·수치·금지어) + LLM Judge 정성 판단으로 품질 검증
- **유저 선택지 제공:** 검증된 패턴 후보를 사용자에게 제시 → 사용자가 패턴 선택 + KPT 기록 완성

---

**WF3 — RAG 기반 리포트 생성**

- **근거 검색 쿼리 생성자:** 사용자 데이터 기반 RAG 검색 쿼리 생성
- **Supabase Vector Store:** pgvector 기반 유사 문서 검색 및 Context 전달
- **리포트 작성자 (Claude Sonnet):** RAG Context + KPT 기록 + 사용자 설정 정보를 결합하여 개인화 리포트 작성
- **리포트 검수자:** 생성된 리포트 품질 검증
- **최종 맞춤형 리포트 발행:** 검수 완료된 리포트를 사용자에게 제공

#### 서비스 전체 워크플로우

| 단계 | 내용 |
|------|------|
| ① 사용 데이터 수집 및 로컬 전처리 | Android UsageStats API로 앱별 사용 시간·실행 횟수 등 수집. 사용 데이터를 snapshot으로 가공하여 시간대별 분석 가능한 데이터셋으로 재구성 |
| ② AI 사용 패턴 분석 | LLM 기반 특이 패턴 감지. KPT를 사용자가 스스로 평가하여 매일의 기록 완성. 체크인 응답(KPT)과 결합하여 맥락 분석 |
| ③ AI 리포트 생성 | 분석된 리포트 생성 결과 변환. 일일/주간 개인화 피드백 제공. 코드 1차 검수, AI 검수자를 통한 2차 검수로 안정된 응답 제공 |
| ④ 게임화 요소 | 프로필/배경 커스텀, 소셜 기능, 업적 달성 시 보상 시스템 → 체크인 동기 부여 |

### 3.4 주요 기능의 구현

#### 데일리 체크인

매일 지정 시간에 푸시 알림을 발송하고, 당일 사용 데이터를 세션 단위로 전처리한 후 AI가 특이 사용 패턴 5가지 후보를 자연어 문장으로 제시한다. 사용자는 KPT 형식으로 패턴을 선택하여 회고를 기록한다.

- 푸시 알림: 매일 사용자가 지정한 시간에 체크인 알림 발송
- 당일 사용 데이터 세션 단위 전처리 후 AI 호출
- AI 자연어 기반 문장 생성 후 5개의 특이 사용 패턴 후보 제시
- K(Keep) / P(Problem) / T(Try) 패턴 선택 및 기록

> **[이미지: 데일리 체크인 UI 화면 — 패턴 목록 화면 / KPT 선택 화면]**

---

#### AI 개인화 리포트

**데일리 리포트**

- 사용자가 선택한 KPT 패턴 / 사용 흐름 / 주 사용 카테고리 정보 제공
- 당일 KPT 패턴과 온보딩 정보를 기반으로 AI 코멘트 생성

> **[이미지: 데일리 리포트 UI 화면 — 사용 흐름 그래프, 주 사용 카테고리, AI 코멘트]**

**주간 리포트**

- 데일리 리포트와 온보딩 정보를 기반으로 생성
- 점수 / 추세분석 / 체크인 참여 횟수 정보 제공
- AI 기반 주간 코멘트 생성
- 데일리/주간 리포트 모두 캘린더에서 날짜별 조회 가능

> **[이미지: 주간 리포트 UI 화면 — 점수 추세 분석, 체크인 참여 현황, AI 주간 코멘트]**

> **[이미지: 캘린더 조회 화면]**

#### 소셜 기능

Google OAuth 기반 계정 연동을 바탕으로 소셜 기능을 제공한다.

- 친구 추가, 삭제, 신청 등 기본 친구 커넥션 기능 (서로 응원 알림 주고받기 — 랜덤 발송)
- 친구 사용자들과 일주일 단위 그룹 챌린지 기능 (체크인 기록, 리포트 조회, 주간 리포트 생성 성공 등)
- 그룹 챌린지 달성 보상으로 골드/경험치를 난이도 및 지속 일수에 따라 차등 지급
- 그룹 상태 화면에서 다른 사용자들의 메인 캐릭터 확인 (같은 바다에서 유영하는 UI로 연대감 제공)

> **[이미지: 소셜 기능 UI 화면 — 친구 목록 / 그룹 챌린지 바다 화면]**
